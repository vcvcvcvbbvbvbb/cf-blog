# 张三的个人博客 - 项目方案说明

## 一、 项目组织结构
项目采用全静态化架构，通过“配置与内容分离”的设计，将所有可变参数抽象至统一配置文件中，实现极速定制：

```text
├── content/               # 内容库：存放 Markdown 格式的文章
├── scripts/               # 自动化工具：包含 prebuild.cjs文章索引生成脚本
├── public/                # 公共资源：存放本地图片(images/)及图标
├── src/                   # 前端核心源码
│   ├── user-config.ts     # 【核心配置】管理姓名、头像、菜单、链接、主题色等
│   ├── components/        # 组件库：Sidebar, PostCard, Notification 等
│   ├── App.tsx            # 应用主入口，处理路由与全局状态
│   └── posts-data.json    # 自动生成的文章索引（构建阶段产出）
├── server.ts              # 服务端入口：支持开发环境与生产环境的高性能服务
├── package.json           # 项目清单：定义构建流程与依赖库
└── README.md              # 面向最终用户的详细使用手册
```

---

## 二、 核心技术亮点

### 1. 极致简化的“零代码”定制 (src/user-config.ts)
针对非开发人员，我们设计了中心化配置系统。用户只需通过文本编辑 `user-config.ts` 中的值，即可实现：
*   **品牌身份**: 一键更换全站标题、博主头像与简介。
*   **动态导航**: 灵活定义侧边栏菜单项与外部社交链接。
*   **视觉统一**: 通过 `THEME_COLOR` 变量，实现全站 UI 高亮色的一键翻新。

### 2. 自动化内容引擎
通过 `scripts/prebuild.cjs` 脚本，在构建过程中自动解析 Markdown 文件的元数据（标题、日期、标签、摘要等），并生成极小的 JSON 索引文件。这种“内容驱动构建”的模式，既保证了撰写的便利性，又实现了页面的秒开体验。

### 3. 多层归档与交互反馈
*   **多维归档结构**: 归档页面支持“年 -> 月 -> 日”三级索引，通过 `reduce` 函数在渲染时自动分组，数据层级清晰。
*   **即时通知系统**: 内置气泡提示组件（Notification），通过 React 状态驱动，为剪贴板复制、分享等操作提供友好的 UI 响应。
*   **全平台响应式**: 采用移动端优先的设计，优化了侧边栏在手机端的唤起逻辑，确保在各种屏幕比例下均有完美的阅读体验。

---

## 三、 关键功能清单
*   **沉浸式阅读**: 干净的中文字体排版，支持代码高亮与 Markdown 全语法实现。
*   **多维内容检索**: 提供根据日期、标签及全文搜索的搜索/过滤功能。
*   **草稿保护机制**: 文章头部设置 `draft: true` 即可在发布时自动隐藏，安全管理待办创作。
*   **交互反馈系统**: 内置气泡提示组件，为“分享链接”、“点赞”等操作提供直观的 UI 反馈。

---

## 四、 结语
本项目致力于打造一个“无需学习代码即可拥有专业级审美”的博客模板。通过对底层逻辑的高度封装，用户可以将精力全部投入到创作中。
