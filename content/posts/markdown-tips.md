## 🖋️ 为什么使用 Markdown？

Markdown 是一种轻量级标记语言，它允许你“易于阅读、易于写作”。对于博主来说，Markdown 最大的优势是**内容与样式分离**，让你专注于文字本身，而排版则交给系统自动处理。

---

## 🚀 核心语法进阶

### 1. 标题与层级
使用 `#` 表示标题。注意：建议从 `##` 开始使用，因为 `H1` 通常被作为文章的标题自动渲染。
- `## 二级标题`：主要章节
- `### 三级标题`：子章节

### 2. 引用与强调
如果你想引用某段话，或者强调某些重点：
> 极简主义不是简单的丢弃，而是有目的地留白。

- **加粗**: `**文字**` -> **粗体**
- *倾斜*: `*文字*` -> *斜体*
- ~~删除线~~: `~~文字~~` -> ~~删除线~~

### 3. 代码高亮
CyberLog 原生支持代码块。使用三个反斜杠 ``` 并在后面跟上语言名称：

```javascript
const welcome = () => {
  console.log("Welcome to CyberLog!");
};
```

## 🎨 优雅排版建议

### 列表的使用
合理使用无序列表 `-` 和有序列表 `1.`，可以极大地提高信息读取速度。

- **不要堆砌文字**: 长段落会让人感到疲惫，适当拆分为短句。
- **善用分割线**: 使用 `---` 来划分不同的讨论模块。
- **图片插入**: 尽量使用清晰、高颜值的图片作为插边图，提升视觉体验。

### 插入图片
在 Markdown 中插入图片非常简单：
`![图片描述](图片链接)`

## 🚀 小贴士
在 CyberLog 中，你可以通过修改 Markdown 文件名来改变文章的 Slug（URL 地址）。建议使用英文或拼音命名，例如 `my-first-post.md`。
